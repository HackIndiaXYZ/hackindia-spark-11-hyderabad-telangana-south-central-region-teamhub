     +-------------+  +-------------+  +-------------+


2.2  REQUEST FLOW (How a user message becomes an AI response)
--------------------------------------------------------------

STEP 1: User types message in Chat.tsx
         |
STEP 2: api.ts sends POST /api/v1/chat with message payload
         |
STEP 3: FastAPI chat.py receives request
         |
STEP 4: Orchestrator analyzes intent, selects agent(s)
         |  - Keyword matching routes to appropriate agent
         |  - e.g. "debug this" -> debugger_agent
         |  - e.g. "write tests" -> tester_agent
         |
STEP 5: Selected agent builds prompt with:
         |  - System prompt (agent's specialty)
         |  - User message
         |  - Memory context (from memory_service)
         |
STEP 6: LLM service calls Ollama (gemma3:1b)
         |  - If Ollama unavailable, falls back to mock responses
         |  - Response streams token by token
         |
STEP 7: Response chunks sent back via WebSocket
         |
STEP 8: Frontend renders each chunk in real-time
         |
STEP 9: Memory service saves useful context for future reference


2.3  AGENT ORCHESTRATION FLOW
------------------------------

User Request
    |
    v
+------------------+
|   Orchestrator   |  <-- orchestrator.py uses LangGraph StateGraph
+------------------+
    |
    |--> Intent Analysis (keyword matching)
    |
    |--> Agent Selection (1 or more agents)
    |
    +--> +------------+
         | Coder      |--> Writes code
         | Debugger   |--> Finds bugs
         | Architect  |--> Designs structure
         | Reviewer   |--> Reviews quality
         | Documenter |--> Writes docs
         | Tester     |--> Creates tests
         | Refactorer |--> Optimizes code
         | Explainer  |--> Explains logic
         +------------+
              |
              v
         +----------+
         | Results  |--> Formatted response sent to user
         +----------+


2.4  CODE VISUALIZATION FLOW
------------------------------

User pastes code
    |
    v
+------------------------+
| trace_generator.py     |  Parses code using Python AST
+------------------------+
    |
    |--> Algorithm detection (algorithm_detector.py)
    |    |  - Bubble sort? Binary search? Recursion? Tree?
    |    |
    |    +--> Generates type-specific trace steps
    |         - Sorting: compare, swap steps
    |         - Searching: compare, narrow steps
    |         - Recursion: call stack, base case steps
    |
    v
+------------------------+
| Trace JSON sent to     |
| Frontend               |
+------------------------+
    |
    v
+------------------------+
| VisualizationPanel     |  Renders animated visualization
| - VisualizationControls|  (play/pause/step)
| - DSAVisualizers       |  (array, tree, graph animations)
| - ExecutionDetails     |  (variable tracking)
| - ProgramOutputPanel   |  (stdout display)
+------------------------+


2.5  WEBSOCKET FLOW (Real-time communication)
-----------------------------------------------

Frontend                           Backend
   |                                 |
   |--- ws://host:port/ws/{id} ----->|  Connection established
   |                                 |
   |--- {"type":"chat", ...} ------->|  User sends message
   |                                 |
   |<-- {"type":"chunk", ...} -------|  Streaming response chunks
   |<-- {"type":"chunk", ...} -------|
   |<-- {"type":"chunk", ...} -------|
   |<-- {"type":"done"} -------------|  Response complete
   |                                 |
   |--- {"type":"ping"} ------------>|  Keep-alive
   |<-- {"type":"pong"} -------------|
   |                                 |
   |    (auto-reconnect on close)    |


2.6  MEMORY SYSTEM FLOW
-------------------------

Save memory:
  User interaction --> memory_service.save() --> TF-IDF vectorize
                                                      |
                                              +-------v--------+
                                              | Vector Store    |
                                              | (in-memory)     |
                                              +-------+--------+
                                                      |
                                              +-------v--------+
                                              | SQLite DB       |
                                              | (persistent)    |
                                              +----------------+

Search memory:
  Query --> TF-IDF vectorize --> Cosine similarity search
                                       |
                               +-------v--------+
                               | Top-K results   |
                               | (ranked by      |
                               |  relevance)     |
                               +----------------+

SECTION 3: KEY FILE DESCRIPTIONS
================================================================================

BACKEND CORE:
  main.py              - Creates FastAPI app, registers 8 routers, sets up
                         WebSocket endpoint, creates DB tables on startup
  config.py            - Loads all settings from .env: database URL, LLM
                         model, JWT secrets, CORS origins
  database.py          - Async SQLAlchemy engine, session factory, table
                         creation
  security.py          - JWT token creation/validation, bcrypt password
                         hashing

BACKEND AGENTS:
  base_agent.py        - Abstract class all agents extend (defines interface)
  orchestrator.py      - THE BRAIN: LangGraph StateGraph that routes requests
                         to agents, chains their execution, collects results
  registry.py          - Maps "coder" -> CoderAgent, "debugger" -> DebuggerAgent
  *_agent.py (x8)      - Each agent's system prompt + LLM invocation logic

BACKEND SERVICES:
  llm_service.py       - Multi-provider LLM client (Ollama/OpenAI/fallback)
  memory_service.py    - TF-IDF vector store + SQLite persistence
  code_runner.py       - Sandboxed Python code execution with tracing
  trace_generator.py   - AST-based algorithm detection + trace generation
  algorithm_detector.py - Pattern-matching algorithm type detection
  websocket_manager.py - WebSocket connection lifecycle management

FRONTEND ENTRY:
  main.tsx             - React root, BrowserRouter, 15 route definitions
  api.ts               - Centralized HTTP client for all API calls
  types.ts             - Shared TypeScript interfaces

FRONTEND STATE:
  useStore.ts          - Zustand store: projects, chats, sidebar state
  websocket.ts         - WebSocket client with auto-reconnect

FRONTEND COMPONENTS:
  Layout.tsx           - App shell with sidebar navigation
  CodeEditor.tsx       - Monaco Editor (VS Code editor in browser)
  Terminal.tsx         - xterm.js terminal emulator
  MarkdownRenderer.tsx - Renders AI markdown responses

FRONTEND VISUALIZATION:
  VisualizationPanel.tsx      - Main container
  VisualizationControls.tsx   - Play/pause/step controls
  DSAVisualizers.tsx          - Array, tree, graph animations
  ExecutionDetails.tsx        - Variable state tracking

FRONTEND PAGES:
  Welcome.tsx          - Futuristic landing page with video
  Dashboard.tsx        - Main hub after login
  Chat.tsx             - AI chat with streaming markdown
  CodeVisualization.tsx - Full-page code visualization
  AgentPage.tsx        - Generic agent interaction page

SECTION 6: QUICK NAVIGATION GUIDE
================================================================================

"I want to add a new API endpoint"
  --> backend/app/api/v1/ (add or edit a route file)
  --> Register in backend/app/main.py

"I want to add a new AI agent"
  --> backend/app/agents/new_agent.py (create agent class)
  --> backend/app/agents/registry.py (register it)
  --> backend/app/api/v1/agents.py (add API endpoint)

"I want to change the UI"
  --> frontend/src/components/ (reusable parts)
  --> frontend/src/pages/ (full pages)

"I want to modify how data is stored"
  --> backend/app/models/ (SQLAlchemy models)
  --> backend/app/schemas/ (Pydantic validation)

"I want to change the LLM provider"
  --> backend/app/services/llm_service.py
  --> backend/.env (MODEL_PROVIDER, MODEL_NAME)

"I want to add a new visualization type"
  --> backend/app/services/trace_generator.py
  --> frontend/src/components/visualization/

"I want to debug the frontend"
  --> Open browser dev tools
  --> Check frontend/src/services/websocket.ts for WS logs

"I want to debug the backend"
  --> Check backend/server.log for FastAPI logs
  --> Run: uvicorn app.main:app --reload --port 8000
